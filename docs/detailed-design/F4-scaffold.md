# [R4] 场景化脚手架需求（当前实现对齐版）

## 范围
- 项目创建流程中的模板与初始化能力
- 脚手架模块当前能力边界

## 功能需求
- 支持在项目创建流程中填写基础参数并创建项目
- 支持在创建后应用蜂群配置（规则文档、配置文件、Skills）
- 支持脚手架模板元数据维护（模板 ID、变量定义、仓库地址）

## 当前实现基线
- `/scaffold` 路由目前是占位页面，尚未承载完整模板创建交互
- 模板注册表当前仅包含 Expo 模板基线
- 模板仓库自动拉取与完整变量替换流程未作为当前交付承诺

## 集成要求
- 前端模块：`features/scaffold`（当前以数据结构定义为主）
- 项目创建页：`features/kanban/ui/CreateProjectPage.tsx`
- 与 `features/swarm-config` 的配置写入流程兼容

## 非目标
- 不保留模板页面视觉描述
- 不保留 UI 资源路径与截图引用
