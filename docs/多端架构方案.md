# Bee Platform 多端架构方案

> 本文档描述 Bee Platform 支持 Web、桌面端、移动端三端的统一架构设计。

---

## 1. 核心定位

### 1.1 产品定位

**Bee Platform 是个人本地工具，不是 SaaS 服务**

- **后端位置**：永远运行在用户电脑上 (`localhost:3847`)
- **数据存储**：SQLite 数据库在本地文件系统
- **无云端**：不部署远程服务器，数据不离开用户电脑
- **访问方式**：用户通过不同界面访问同一本地后端

### 1.2 整体架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              用户电脑                                         │
```

### 1.3 关键设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| **部署方式** | 个人本地工具 | 数据隐私，无需服务器成本 |
| **后端架构** | 单一 Rust HTTP Server | 统一代码库，跨平台复用 |
| **前端构建** | Next.js + rewrites | Web 开发时代理到本地后端（解决 CORS） |
| **移动端定位** | 局域网/公网客户端 | 访问用户电脑上的本地服务 |

## 2. 三端实现方案

### 2.1 Web 端（浏览器访问）

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                  │
│  用户浏览器                                                       │
│      │                                                           │
│      ▼                                                           │
│  Next.js (localhost:3000)                                        │
│      │                                                           │
│      │ rewrites (解决 CORS)                                      │
│      ▼                                                           │
│  Rust HTTP Server (localhost:3847)                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**部署方式**：
- 用户在电脑上启动 Rust HTTP Server (3847)
- 用户在浏览器访问 `localhost:3000`
- Next.js rewrites 将 `/api/*` 代理到 `localhost:3847/api/*`

**特点**：
- ✅ 无需安装，浏览器直接访问
- ✅ 前端可独立更新
- ⚠️ 需要本地运行后端服务

```
│  └───────────────────────────────────────────┘  │
│                      │                          │
│                      ▼                          │
│  ┌───────────────────────────────────────────┐  │
│  │  Rust 后端 (内嵌在 App 中)                │  │
│  │  - localhost:3847                         │  │

│  │  - 本地 Git 操作                          │  │
│  └───────────────────────────────────────────┘  │
```

**技术栈**：

- ✅ 开发时热重载
- ✅ 直接访问本地文件系统

- ✅ 性能最佳

**支持平台**：
- Windows (x64, ARM64)
- macOS (Intel, Apple Silicon)
- Linux (x64, ARM64)

```
│  │  - 发起 Git 操作请求（commit、push、merge）                │ │
│  │  - 查看日志、状态、历史                                    │ │
│  │                                                           │ │
│  │  ❌ 不做的：                                               │ │
│  │  - 本地执行 git 命令                                       │ │
│  │  - 本地运行 opencode/claude CLI                           │ │
│  │  - 本地文件系统操作                                        │ │
│  │  - 本地启动 HTTP Server                                   │ │
│  └───────────────────────────────────────────────────────────┘ │
│                            │                                    │
│                            │ 局域网 HTTPS / 公网 HTTPS           │
│                            ▼                                    │
│                  ┌─────────────────────────────────────┐        │
│                  │    用户电脑上的 Rust 后端             │        │
│                  │                                     │        │
│                  │  局域网: 192.168.x.x:3847           │        │
│                  │  公网: user-domain.com:3847         │        │
│                  │                                     │        │
│                  │  真正的执行在这里:                   │        │
│                  │  - git 命令执行                      │        │
│                  │  - opencode CLI 运行                 │        │
│                  │  - 文件读写                          │        │
│                  │  - 数据库操作                        │        │
```

**访问方式**：

| 方式 | 地址 | 说明 |
|------|------|------|
| **局域网** | `http://192.168.x.x:3847` | 用户在同一网络下 |
| **公网** | `https://user-domain.com:3847` | 用户开放端口到公网 |

**技术栈**：

| 组件 | 技术 | 说明 |
|------|------|------|
| 框架 | Tauri v2 (Mobile) | 仅作为 WebView 容器 |
| 前端 | Next.js | 复用 Web 端代码 |
| 后端 | **用户电脑上的 Rust 服务** | 通过网络访问 |
| WebView | iOS: WKWebView / Android: WebView | - |

**移动端 Rust 代码边界**：

| ✅ 可以包含 | ❌ 不能包含 |
|------------|------------|
| Tauri 插件（通知、分享等） | AgentProcessManager |

| 极简的本地配置存储 | HTTP Server (http_server.rs) |
| | SQLite 本地数据库（业务数据） |
| | git2 库编译 |

**特点**：
- ✅ 查看任务状态和结果
- ✅ **可以发起任何操作请求**（创建任务、Git 操作等）
- ✅ 推送通知
- ✅ 安装包体积小（无本地后端）
- ⚠️ 真正执行在用户电脑上完成
- ⚠️ 需要网络连接（局域网或公网）

**支持平台**：
- iOS 14+
- Android 7.0+ (API 24+)

### 2.4 前端能力 vs 后端执行（重要澄清）

> **关键理解**：Web/移动端「可以操作」，但「执行在用户电脑上」

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           用户操作（前端）                               │
│                                                                         │
│   "我点击了「提交代码」按钮"                                            │
│   "我输入了 prompt 发送给 Agent"                                       │
│   "我创建了新的 workspace"                                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
                              │
                              │ HTTPS 请求
                              ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                         真正执行（用户电脑后端）                         │
│                                                                         │
│   用户电脑上的 Rust HTTP Server 执行：                                  │
│   - git add . && git commit -m "..."                                   │
│   - git push origin main                                                │
│   - opencode --prompt "..." --workspace /path/to/ws                    │
│   - 读写文件系统                                                        │
│   - 更新数据库                                                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

| 操作 | Web 端 | 移动端 | 桌面端 |
|------|--------|--------|--------|
| 点击「提交」按钮 | ✅ 可以 | ✅ 可以 | ✅ 可以 |
| git 命令执行 | 用户电脑 | 用户电脑 | **本机** |
| 运行 opencode | 用户电脑 | 用户电脑 | **本机** |
| 读写文件 | 用户电脑 | 用户电脑 | **本机** |

**结论**：三端 UI 能力相同，区别在于「访问方式」

## 3. API 调用规范（重要）

### 3.1 核心原则

**统一使用 HTTP API**：所有业务逻辑通过 HTTP API 调用后端，确保所有端（Web、Desktop、Mobile）功能一致。

```
┌─────────────────────────────────────────────────────────────┐
│                    前端应用                                  │
│         (Web / Desktop Tauri / Mobile Tauri)                 │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│              HTTP API (统一入口)                            │
│              localhost:3847 (或局域网/公网)                  │
├─────────────────────────────────────────────────────────────┤
│ • 项目/任务 CRUD                                             │
│ • 蜂群配置管理                                               │
│ • Git 操作 (rebase, merge, commit, push)                     │
│ • Worktree 管理                                              │
│ • 设置管理                                                   │
│ • Agent 控制 (启动/停止/状态查询)                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Tauri IPC 使用边界

**Tauri IPC 仅用于系统级功能**，与业务逻辑解耦：

| 功能类别 | 使用方式 | 示例 |
|---------|----------|------|
| **业务逻辑** | HTTP API | CRUD、Git、配置文件读写 |
| **系统对话框** | Tauri IPC | 文件选择器、文件夹选择器 |
| **系统通知** | Tauri IPC | 系统托盘通知、原生通知 |
| **窗口控制** | Tauri IPC | 创建/关闭窗口、窗口状态 |
| **原生功能** | Tauri IPC | 剪贴板、屏幕捕获 |

**禁止事项**：
- ❌ 业务代码使用 `invoke()` 调用后端
- ❌ 通过 Tauri IPC 执行 Git 操作
- ❌ 通过 Tauri IPC 读写配置文件
- ❌ 通过 Tauri IPC 管理设置

**正确做法**：
- ✅ 所有业务逻辑通过 `fetch('/api/...')` 调用
- ✅ Tauri IPC 只保留给无法通过 HTTP 实现的系统功能

### 3.3 API 端点列表

| 分类 | 端点 | 说明 |
|------|------|------|
| **项目** | `GET/POST/PUT/DELETE /api/projects` | 项目 CRUD |
| **任务** | `GET/POST/PUT/DELETE /api/tasks` | 任务 CRUD |
| **蜂群** | `GET/POST/PUT/DELETE /api/swarms` | 蜂群管理 |
| **蜂群绑定** | `GET/POST/PUT/DELETE /api/projects/:id/swarm-bindings` | 项目蜂群绑定 |
| **Git** | `POST /api/git/*` | Git 操作 |
| **Worktree** | `GET/POST /api/worktree/*` | Worktree 管理 |
| **设置** | `GET/PUT /api/settings` | 全局设置 |
| **Agent** | `POST /api/agents/*` | Agent 控制 |

## 4. API 调用配置

### 4.1 核心逻辑

所有平台通过 [`api-config.ts`](features/agent-execution/lib/api-config.ts) 自动选择正确的后端地址：

```typescript
function getApiBase(): string {
  // 1. 手动指定的环境变量优先
  if (process.env.NEXT_PUBLIC_API_BASE) return ...

  // 2. Tauri 桌面端 → 本地 Rust HTTP Server
  if (isTauriDesktop()) {
    return 'http://127.0.0.1:3847'
  }

  // 3. Tauri 移动端 → 用户配置的地址（局域网或公网）
  if (isTauriMobile()) {
    // 优先使用用户配置的地址，否则使用局域网检测
    return getUserConfiguredApiBase() || detectLocalNetworkApiBase()
  }

  // 4. Web → 使用相对路径，通过 rewrites 代理到 3847
  return ''
}
```

### 4.2 Web 端 Rewrites

[`apps/web/next.config.ts`](apps/web/next.config.ts)：

```typescript
async rewrites() {
  // 始终需要 rewrites，因为后端在本地
  return [
    {
      source: '/api/:path*',
      destination: 'http://localhost:3847/api/:path*',
    },
  ]
}
```

**注意**：与文档旧版本不同，rewrites 在生产环境也需要，因为：
1. 后端始终在本地 localhost:3847
2. 浏览器直接访问会有 CORS 问题
3. Next.js rewrites 提供透明的代理层

### 4.3 WebSocket 连接

```typescript
function getWsBase(): string {
  // 1. 手动指定的环境变量优先
  if (process.env.NEXT_PUBLIC_WS_BASE) return ...

  // 2. Tauri 桌面端 → 本地 Rust
  if (isTauriDesktop()) {
    return 'ws://127.0.0.1:3847'
  }

  // 3. Tauri 移动端 → 用户配置的地址
  if (isTauriMobile()) {
    return getUserConfiguredWsBase() || detectLocalNetworkWsBase()
  }

  // 4. Web → 直接连接 3847
  // Next.js API Routes 不支持 WebSocket
  return 'ws://127.0.0.1:3847'
}
```

## 8. 移动端特殊处理

### 8.1 局域网服务发现

移动端需要自动发现局域网内的后端服务：

```typescript
// features/mobile/lib/network-discovery.ts
export async function discoverLocalServer(): Promise<string | null> {
  // 扫描常见局域网段
  const subnets = ['192.168.1.', '192.168.0.', '10.0.0.']

  for (const subnet of subnets) {
    for (let i = 1; i <= 254; i++) {
      const ip = `${subnet}${i}`
      try {
        const response = await fetch(`http://${ip}:3847/api/health`, {
          signal: AbortSignal.timeout(100)
        })
        if (response.ok) return `http://${ip}:3847`
      } catch {
        // 继续尝试
      }
    }
  }

  return null
}
```
